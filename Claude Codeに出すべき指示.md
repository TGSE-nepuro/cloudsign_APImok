# Claude Code 再起動後に出すべき指示

## 起動方法
ターミナルで以下を実行：
```bash
cd /Users/edamoto/cloudsign_APImok
claude
```
起動後、`CLAUDE.md` が自動で読み込まれプロジェクトのコンテキストが復元されます。

---

## セッション開始時の指示（コピー＆ペーストで使用可）

```
クラウドサインAPI mokプロジェクトの開発を再開します。
CLAUDE.mdの内容を確認の上、組込み署名機能の開発をエージェントチームを
使って進めてください。開発開始前に作業計画を提示してください。
```

---

## 利用できるスキル（スラッシュコマンド）

| コマンド | 用途 |
|---|---|
| `/commit` | 変更内容を確認してgitコミットを作成 |
| `/test <対象>` | TDDに従いテストコードを作成・レビュー・実行 |
| `/review <対象>` | バグ・セキュリティ・脆弱性・アクセシビリティの観点でコードレビュー |

---

## エージェントチームズの起動例

```
エージェントチームを作成して組込み署名機能を並行開発してください。
担当を以下のように分けてください：
- 1人目：送信画面の改修（通常送信・SMS認証・簡易認証の3種類に分割）
- 2人目：同意用マイページ画面の新規作成
- 3人目：案件一覧画面への組込み署名送信済み表示の追加
各担当はTDDで進め、作業都度 作業記録.txt に記録してください。
```

---

## 参照ファイル
- `CLAUDE.md`：プロジェクト全体の指示・ルール・追加要件
- `指示書.txt`：元の開発指示書（全要件・制約・開発方針）
- `作業記録.txt`：作業履歴
- `docs/`：組込み署名の仕様書PDF（git管理外・機密）
